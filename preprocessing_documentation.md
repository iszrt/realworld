# Data Preprocessing Documentation

**Project:** CITC Department University Course Timetabling using ETFCSA-TSD
**Date:** March 2026
**Purpose:** Document all data quality issues found in the original CITC schedule data and the preprocessing steps applied to resolve them, for reproducibility and thesis defense justification.

---

## 1. Data Flow Overview

```
CITC SCHEDULING.xlsx          (original department schedule — source of truth)
        |
        v
  preprocess.py               (normalization & cleaning)
        |
        +---> cleaned/CITC_SCHEDULING_CLEANED.xlsx   (cleaned initial schedule)
        +---> cleaned/Algorithm_Input_Cleaned.xlsx    (cleaned optimizer input)
                    |
                    v
              app.py / tsd.py  (ETFCSA-TSD optimization)
                    |
                    v
              Optimized Schedule (Excel + ITC 2019 XML)
```

### Original Files (NOT modified)
| File | Description |
|------|-------------|
| `CITC SCHED/CITC SCHEDULING.xlsx` | The original department schedule with 100 section sheets, 755 class meeting entries |
| `Structured_Data_Algorithm_Input_Refined.xlsx` | A manually extracted version created by a team member, stripped of Day/Time/Room for optimizer input (488 entries) |

### Cleaned Files (Generated by `preprocess.py`)
| File | Description |
|------|-------------|
| `cleaned/CITC_SCHEDULING_CLEANED.xlsx` | Normalized version of the original schedule (same structure, 100 sheets) |
| `cleaned/Algorithm_Input_Cleaned.xlsx` | Properly derived optimizer input with all 755 class meetings preserved |

| 23-105 LRC BLDG | 23-105 LRC BLDG(LEC) | Missing label |

Duplicate room names cause the optimizer to treat the same physical room as two separate rooms. This means it could assign two classes to the same room at the same time without detecting a conflict, because it sees "09-301 (CADD Lab)" and "09-301 (CADD Lab 1)" as different rooms.

**Result:** 77 room strings reduced to 69 actual rooms.

### 2.5 Lost Class Meetings in Original Input File

The most critical issue: the original `Structured_Data_Algorithm_Input_Refined.xlsx` had **488 entries** while the CITC schedule has **755 class meetings**.

**Root cause:** When extracting the optimizer input, multi-day meetings were collapsed. For example, CS121 for section CS1A meets on both Monday and Friday — the CITC schedule has 2 rows, but the input file had only 1 (or 2 due to a typo creating a false duplicate).

| Metric | Original Input | Cleaned Input |
|--------|---------------|---------------|
| Total class meetings | 488 | 755 |
| Meetings recovered | — | +267 |

The cleaned input now includes a `Meeting_Index` column to distinguish multi-day meetings of the same subject:

| Meeting_Index | Count | Meaning |
|--------------|-------|---------|
| 1 | 484 | First (or only) meeting of the week |
| 2 | 266 | Second meeting of the week |
| 3 | 3 | Third meeting (rare, likely 3-hour labs split across days) |
| 4-5 | 2 | Edge cases |

### 2.6 Time Format Inconsistencies

The original CITC schedule has **76 different time format strings**, including:

- Standard: `07:00 AM - 08:00 AM` (most common)
- Bare 24h: `07:30:00`, `17:30:00` (some CS1D, TCM sections)
- Malformed: `12:00:00 AM - 3:30 PM` (missing leading zero)
- Variable durations: 1-hour, 1.5-hour, 2-hour, and 3-hour blocks

The preprocessing normalizes time strings to a consistent format where possible. The bare 24-hour times (where end time is unavailable) are converted to `HH:MM AM/PM` format.

## 3. Preprocessing Script

**File:** `preprocess.py`

**What it does:**
1. Reads the original `CITC SCHEDULING.xlsx` (does NOT modify it)
2. Applies normalization mappings for instructors, subjects, and rooms
3. Normalizes time format strings
4. Outputs cleaned versions to the `cleaned/` directory
5. Derives a proper optimizer input with all 755 class meetings preserved

**How to run:**
```bash
python preprocess.py
```

**Normalization approach:** Simple dictionary lookup — each known variant maps to its canonical form. This is deterministic, transparent, and easily auditable.

## 4. Online Class Detection

### 4.1 Rationale

In the original CITC schedule, 197 out of 755 class meetings (26%) have no room assigned. Investigation revealed these are **online/virtual class meetings** or PE classes held at outdoor facilities with no formal room code.

**Subjects with highest no-room counts:**

| Subject | No-Room Count | Likely Reason |
|---------|--------------|---------------|
| PICPE | 35 | Online/self-paced |
| STS | 27 | Online lecture component |
| ArtApp | 27 | Online lecture component |
| PATH FIT 2 | 24 | PE (gym/outdoor, no fixed room) |
| PATH FIT 4 | 14 | PE (gym/outdoor, no fixed room) |
| Rizal | 13 | Online |
| PurCom | 9 | Online |

### 4.2 Classification Logic

A class meeting is classified as **online** if its Room field is NaN/empty in the original schedule. This is consistent with CITC department practice.

**Multi-day class patterns observed:**
- **220 multi-day groups** have rooms for ALL meetings (fully face-to-face)
- **50 multi-day groups** have NO rooms for any meeting (fully online)
- **6 multi-day groups** have mixed assignment (one meeting face-to-face, one online -- hybrid classes)

### 4.3 Impact on the Optimizer

The cleaned input now includes a `Mode` column:

| Mode | Count | Optimizer Behavior |
|------|-------|--------------------|
| face-to-face | 553 | Needs room + timeslot assignment |
| online | 202 | Needs timeslot assignment only (skip room) |

Both modes participate in **instructor conflict** and **section conflict** checking. Only face-to-face meetings participate in **room conflict** and **room type** checks.

After preprocessing, the remaining same-last-name instructor groups were manually verified to be **different individuals** (different first names, different subjects). No false merges were introduced.

1. Confirming all 100 sections are preserved
